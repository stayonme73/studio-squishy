## Core principle

**The Studio recommends. The client decides.**

Those are two different things. The Studio is not saying *"This is what you have to buy."* It is saying *"Based on what you told us, this is what we'd do if we were building this project. You're free to adjust it."*

That balance — trusted advisor plus customer choice — is a defining characteristic of The Studio.

## Business rule: Recommendation, Not Direction

The Recommendation Engine provides professional recommendations based on the client's Discovery responses.

Recommendations are intended to **guide, not restrict**.

Clients may:

- Accept the recommended Studio Plan
- Remove recommended Studio Services
- Substitute equivalent Studio Services
- Add additional Studio Services
- Build a custom Studio Plan using approved Studio Services

The Recommendation Engine must **never** force a client to purchase recommended services.

Its role is to provide professional guidance while preserving customer choice.

This principle applies to **every** recommendation generated by The Studio.

## Architecture alignment

| Layer | Role under this rule |
|-------|----------------------|
| Service Catalog | Defines approved Studio Services — the menu clients may choose from |
| Recommendation Engine | Scores Discovery answers and **recommends** — never locks or purchases |
| Discovery Summary Model | Maps recommendations into customer-facing copy (titles, Why?, investment) |
| Project Summary | Presents Our Recommendation + Customize Your Studio Plan — client adjusts plan |
| Studio Plan Review | Remove, substitute, add services — customer choice preserved |

The engine may rank, include, or flag services. Downstream UI must always allow the client to change the plan before confirmation.
